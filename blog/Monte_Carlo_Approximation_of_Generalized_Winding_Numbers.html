Monte Carlo Approximation of Generalized Winding Numbers for Possibly Non-Manifold and Open Meshes

Author: Iyad Ahmed

Abstract

Generalized Winding Numbers (GWNs) are really useful for determining if a point is inside a polygon and various other uses, however, calculating exact GWN values has O(n2) time complexity. This article presents a Monte Carlo method for approximating GWN values in O(log(n)) time.

Introduction

O(n2) method for calculating GWN

In R2, the GWN of a point p with respect to a polygon P is defined as the sum of the signed angles at the point for every triangle that is formed using the point and every edge in the polygon (edge direction matters).

In R3, solid angles at the point for every tetrahedron that is formed using the point and every triangle of the mesh are summed instead, in this case the winding order of the vertices of every one of those triangles matter.

Alternative methods for calculating GWN in 2D and 3D for closed surfaces

There are two main methods:

The even-odd rule

In the even-odd rule method, we intersect a ray from the query point into the mesh and counts the number of intersections, if the number of intersections is odd, then we consider the point to be inside the mesh, if it is even, then we consider the point to be outside.

The non-zero rule

In the non-zero rule, the winding order of mesh or polygon matters, because we intersect a ray from the query point, and at every intersection, we compare the direction of the ray with the direction of the normal of the edge, if mesh is 2D, or triangle if mesh is 3D, if those two directions have a positive dot product, then we give this intersection a value of +1, if the dot product is negative, we give the intersection a value of -1, summing all the intersection values results in either a zero sum or a non-zero sum, points that have a non-zero sum are considered to be inside the mesh, otherwise they are considered to be outside.
